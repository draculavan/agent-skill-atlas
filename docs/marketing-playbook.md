# Marketing Playbook

This playbook keeps growth focused on useful feedback, not empty launch noise.

## Positioning

One-line promise:

> A local-first visual atlas for your AI agent skills.

Target user:

> Agent power users who have accumulated enough local skills and plugins that
> they need a map.

Core privacy promise:

> It scans metadata, not private skill bodies.

## Content Pillars

1. **Local-first privacy**
   - Metadata-only scanning.
   - No upload.
   - Relative paths and redaction.

2. **Agent skill management**
   - Find what skills exist.
   - Spot duplicate or under-documented skills.
   - Summarize plugin packs.

3. **Route preview**
   - Preview likely task-to-skill matches.
   - Keep execution separate from discovery.

4. **Open-source compatibility**
   - Ask users for sanitized folder layouts.
   - Build support around real local setups.

## Weekly Operating Loop

Every week:

- Review stars, forks, watchers, issues, and discussions.
- Identify the most useful next compatibility or privacy improvement.
- Draft one short English update and one short Chinese update.
- Propose one GitHub-native action such as an issue, discussion reply, or README
  improvement.
- Do not post to third-party communities or submit external PRs without explicit
  user approval.

## Launch Channels

Start small:

- GitHub Discussions.
- Personal Chinese/English dev circles.
- Agent skills communities.
- Local-first developer tool communities.

Wait before bigger channels:

- Product Hunt.
- Broad Reddit/Hacker News-style posts.
- Awesome-list PRs.

## Success Signals

Early signals:

- 3-5 sanitized folder layout reports.
- 1-2 privacy review comments.
- First external issue or discussion.
- Users can install from GitHub and open the demo without help.

Stronger signals:

- Support for multiple real skill layouts.
- Demo GIF improves README conversion.
- Small but recurring discussions around taxonomy and privacy.
